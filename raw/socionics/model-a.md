# Model A — The 8 Functions of Information Metabolism

Model A is the model of human information metabolism. When filled with specific information aspects, it forms the type of information metabolism (TIM), i.e., the socionic type of a person. Named after the creator of socionics, Lithuanian sociologist Aušra Augustinavičiūtė.

## Function Properties

Functions are defined by combinations of seven properties that determine how information is perceived, processed, and assimilated:

| Property | Description |
|----------|-------------|
| **Mental/Vital** | Consciousness (mental) or subconscious (vital) processing |
| **Acceptance/Production** | How information is received vs. created |
| **Strong/Weak** | Ability to handle information under pressure |
| **Valued/Unvalued** | Whether the function is important to the person |
| **Inert/Contact** | How the person engages with this information |
| **Loaded/Unloaded** | Energy expenditure when processing |
| **Template/Situational** | Fixed patterns vs. flexible adaptation |

## The 8 Functions

| Function | Name | Block | Properties |
|----------|------|-------|------------|
| **1** | Base (Leading) | Ego | Strong, Conscious, Valued |
| **2** | Creative | Ego | Strong, Conscious, Valued |
| **3** | Role | Super-Ego | Weak, Conscious, Unvalued |
| **4** | Vulnerable (PoLR) | Super-Ego | Weak, Conscious, Unvalued |
| **5** | Suggestive | Super-Id | Weak, Unconscious, Valued |
| **6** | Activation | Super-Id | Weak, Unconscious, Valued |
| **7** | Ignoring | Id | Strong, Unconscious, Unvalued |
| **8** | Demonstrative | Id | Strong, Unconscious, Unvalued |

## Function Descriptions

### Base Function (1st)
- Always active, drives perception
- Primary cognitive filter
- Most comfortable discussing this information
- Others recognize them as expert

### Creative Function (2nd)
- Produces new based on what base accepts
- More situational, task-oriented
- Can rebuild model for current needs
- Less rigid than base, more adaptive

### Role Function (3rd)
- Information processed under social demand
- Internal map held tensely
- Modeling because "that's how it should be"
- Unnatural, effortful application

### Vulnerable Function (4th / PoLR)
- Information processing breaks down under pressure
- Painful topic, avoided if possible
- Can understand fragments but not stable modeling
- Under stress, may completely lose capability

### Suggestive Function (5th)
- Doesn't want to be main author of this information
- Values when partner provides missing structure
- Unconsciously searches for this information from outside
- Feels enhanced when appropriate information is provided

### Activation Function (6th)
- Doesn't always immediately build model
- Energized when gently drawn into it
- Responds to support and acceleration
- Can develop rapidly with appropriate partner

### Ignoring Function (7th)
- Often sees this information quite strongly
- Doesn't want to make it center of interaction
- Can use if needed, doesn't live in it
- Will apply if serves purpose but won't prioritize

### Demonstrative Function (8th)
- Casual, automatic application
- No special emphasis or effort
- Almost unconscious competence
- Provides background support without being focal
