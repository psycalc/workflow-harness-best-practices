# Typology Code Conventions

This page defines how the wiki writes type codes across English, Russian, and Ukrainian pages.

## Rule

Use the code of the page language as the primary code. Add other-language equivalents in parentheses when cross-language reliability matters.

Example:

- English page: `SLI + ELVF + EPNF` (RU: `СЛИ + ЭЛВФ + ВПНБ`; UK: `СЛІ + ЕЛВФ + Вч-Ми-Тп-Мб`)
- Russian page: `СЛИ + ЭЛВФ + ВПНБ` (EN: `SLI + ELVF + EPNF`; UK: `СЛІ + ЕЛВФ + Вч-Ми-Тп-Мб`)
- Ukrainian page: `СЛІ + ЕЛВФ + Вч-Ми-Тп-Мб` (EN: `SLI + ELVF + EPNF`; RU: `СЛИ + ЭЛВФ + ВПНБ`)

Do not mix language-specific codes inside the same sentence unless the sentence explicitly gives equivalents.

## Socionics

Socionics type codes should also be localized where the page language has an established local abbreviation. Keep the Latin code in parentheses for cross-language lookup.

| Meaning | English | Russian | Ukrainian |
|---------|---------|---------|-----------|
| Sensory Logical Introvert | SLI | СЛИ (SLI) | СЛІ (SLI) |

MBTI-like analogues may be shown for orientation, but they are **not equivalent systems**. Socionics `SLI` is often mapped approximately to `ISTP` in MBTI terms and to legacy Socionics notation `ISTp`.

| Socionics | Russian | Ukrainian | Legacy Socionics | MBTI-like analogue |
|-----------|---------|-----------|------------------|--------------------|
| SLI | СЛИ | СЛІ | ISTp | ISTP, approximate only |

Information elements may be shown bilingually where useful:

| Latin | Russian | Ukrainian |
|-------|---------|-----------|
| Si | БС | БС |
| Te | ЧЛ | ЧЛ |
| Ni | БИ | БІ |
| Fe | ЧЭ | ЧЕ |
| Ne | ЧИ | ЧІ |
| Fi | БЭ | БЕ |
| Se | ЧС | ЧС |
| Ti | БЛ | БЛ |

## Temporistics

English uses `N` for Present/Now to avoid collision with `P` = Past.

Russian uses the established one-letter Cyrillic codes.

Ukrainian uses two-letter project codes because `Минуле` and `Майбутнє` both begin with `М`.

| Aspect | English code | Russian code | Ukrainian code |
|--------|--------------|--------------|----------------|
| Eternity | E | В | Вч |
| Past | P | П | Ми |
| Present / Now | N | Н | Тп |
| Future | F | Б | Мб |

Composite example for Eternity-Past-Present-Future:

| Language | Code |
|----------|------|
| English | EPNF |
| Russian | ВПНБ |
| Ukrainian | Вч-Ми-Тп-Мб |

## Position Codes

When writing positions, use language-local aspect codes:

| Meaning | English | Russian | Ukrainian |
|---------|---------|---------|-----------|
| 1st Eternity | 1E | 1В | 1Вч |
| 2nd Past | 2P | 2П | 2Ми |
| 3rd Present | 3N | 3Н | 3Тп |
| 4th Future | 4F | 4Б | 4Мб |

This avoids ambiguity between systems. For example, English `4F` can mean **4th Physics** in Psychosophy or **4th Future** in Temporistics, so the surrounding system must be named explicitly when ambiguity is possible.
