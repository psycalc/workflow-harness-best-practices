# ESCO Typology Mapping

## Overview

ESCO is useful as a **job/skill taxonomy**, while typologies are useful as **fit hypotheses**. The mapping is not one-to-one; it works by converting ESCO occupations into work features, then mapping those features to typological signals.

## Work Features

Normalize each ESCO occupation into a small set of stable features:

- `tech-heavy`
- `people-heavy`
- `structure-heavy`
- `stress-heavy`
- `hands-on`
- `analysis-heavy`
- `coordination-heavy`
- `autonomy-high`
- `long-horizon`
- `mobility-high`

## Typology Signals

### Socionics

Use Socionics for:

- information processing style
- technical vs people-facing preference
- tolerance for ambiguity and change
- communication and coordination style

### Psychosophy

Use Psychosophy for:

- how a person organizes action
- response to pressure and authority
- preference for support vs drive vs control
- tolerance for physical discomfort and material demands

### Temporistics

Use Temporistics for:

- time horizon
- stability vs transition preference
- meaning/orientation over time
- routine vs reorganization fit

## Practical Role Families

### Strong Technical Fit

- `tech-heavy`
- `structure-heavy`
- `autonomy-high`
- `analysis-heavy`

Likely fits:
- Socionics: Delta / Gamma technical types
- Psychosophy: types with strong Logic + lower sensitivity to constant pressure
- Temporistics: types that can sustain a long operational horizon

### Strong Communications Fit

- `people-heavy`
- `coordination-heavy`
- `stress-heavy`
- `autonomy-medium`

Likely fits:
- Socionics: types good at information exchange and coordination
- Psychosophy: strong Emotion + Logic combinations
- Temporistics: types that can carry continuity and relay direction

### Strong UAV / Recon Fit

- `tech-heavy`
- `analysis-heavy`
- `mobility-high`
- `autonomy-high`

Likely fits:
- Socionics: technical-observational profiles
- Psychosophy: action profiles that like concrete systems but not heavy coercion
- Temporistics: role types that handle changing conditions and movement

## Example: DevOps

Typical ESCO features:

- `tech-heavy`
- `structure-heavy`
- `analysis-heavy`
- `autonomy-high`
- `long-horizon`
- `coordination-heavy`

Typical typology implications:

- Socionics: strong match for practical systems work
- Psychosophy: good if logic and structured action are strong, weak if the role requires constant status pressure
- Temporistics: good if the person can sustain continuity and process flow
